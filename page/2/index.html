事件循环

发表于 2020-05-15

什么是事件循环

事件循环，实际上是浏览器执行任务的机制。众所周知，js是单线程的，但是有时候如果要进行一些需要等待的操作，也就是异步操作，不可能一直阻塞等着结果的返回。所以就出现了事件循环。

事件循环的机制

首先，script中执行的代码分为同步和异步。同步代码直接执行，异步代码要在满足条件后被扔到相应的task queue中。

在当前的执行栈为空后，浏览器会先去检查微任务队列，并将其中的任务执行。微任务队列为空后，去检查宏任务队列，如果有任务就执行。

宏任务&微任务

对于每一轮事件循环来说，是从宏任务队列中取出一个宏任务执行，再依次逐步执行微任务队列中的所有微任务。

拓展-node中的事件循环

![image-20200520111252990](/Users/EllieWxy/Library/Application Support/typora-user-images/image-20200520111252990.png)

node中的事件循环分为了六个阶段，顺序是从外部输入数据开始，之后poll-> check->close callbacks -> timers -> I/O callbacks -> idle,prepare-> poll

每个阶段都有它负责的事件。

  • timers 阶段：这个阶段执行timer（setTimeout、setInterval）的回调

  • I/O callbacks 阶段：处理一些上一轮循环中的少数未执行的 I/O 回调

  • idle, prepare 阶段：仅node内部使用

  • poll 阶段：获取新的I/O事件, 适当的条件下node将阻塞在这里

  • check 阶段：执行 setImmediate() 的回调

  • close callbacks 阶段：执行 socket 的 close 事件回调

timers

执行定时器（setTimeout,setInterval)

poll

主要干两件事

  1. 如果定时器已经超时，就返回定时器执行相应的回调函数。
  2. 执行I/O回调
    • （所以大多数的I/O会在poll阶段执行，I/O callback负责执行上轮未执行的少部分回调）
  3. 如果没有要处理的定时器
    • poll不为空的话，就执行poll中的回调函数
    • poll为空
      • 存在setImmediate，从poll阶段进到check阶段
      • 不存在setImmediate，在poll阶段等待直到有事件为止
check

执行setImmediate

nextTick

在上一个阶段结束后，下一个阶段执行前进行执行

node 11 vs node 10

在node 11的机制中，执行完一个宏任务，就会去检查微任务队列并清空。

而在node10以前的机制中，需要执行完一队宏任务，才会去检查微任务队列并清空。

匹配语句

Left join,Right jon,Innerjoin

left join（左联接）：返回左表中的所有记录以及和右表中的联接字段相等的记录。

相当于左表+右表中的等值部分

right join（右联接）：返回右表中的所有记录以及和左表中的联接字段相等的记录。

inner join（等值联接）：只返回两个表中联接字段相等的记录。

相当于两个表的交集

SQL truncate 、delete与drop区别

  • truncate、delete只删除数据不删除表的结构，drop不仅要删除数据还要删除表的结构

sql索引

优点

  • 通过创建唯一性索引，可以保证数据库表中每一行数据的唯一性。
  • 可以大大加快数据的检索速度，这也是创建索引的最主要的原因。
  • 可以加速表和表之间的连接，特别是在实现数据的参考完整性方面特别有意义。
  • 在使用分组和排序子句进行数据检索时，同样可以显著减少查询中分组和排序的时间。
  • 通过使用索引，可以在查询的过程中，使用优化隐藏器，提高系统的性能。

缺点

  • 创建索引和维护索引要耗费时间，这种时间随着数据量的增加而增加。
  • 索引需要占物理空间，除了数据表占数据空间之外，每一个索引还要占一定的物理空间，如果要建立聚簇索引，那么需要的空间
    就会更大。
  • 当对表中的数据进行增加、删除和修改的时候，索引也要动态的维护，这样就降低了数据的维护速度。

响应式布局是什么

响应式布局就是实现不同屏幕分辨率的终端上浏览网页的不同展示方式，即在电脑、手机、平板上会有不同的展示页面。

传统的布局方式是根据不同的尺寸写不同的页面，响应式布局要求一个网站一套就可以

响应式布局的实现方案

1.设置meta标签

表示使用设备的宽度作为视图宽度

2.设置媒体查询

在屏幕宽度小于980px时会使用这一套css

3.使用vh/vw

vh/vw是相对于视窗的尺寸

4.使用百分比

在一些情况下容易炸（要慎重使用）

5.使用一些新特性或框架

例如

  • Flex弹性布局，兼容性较差
  • Grid网格布局，兼容性较差
  • Columns栅格系统，往往需要依赖某个UI库，如Bootstrap

js内存泄漏

发表于 2020-05-14 | 分类于 javascript

什么是内存泄漏

不在用到的内存，没有及时释放，就是内存泄漏

js的垃圾回收机制

  • 引用计数
    • 计算这个值被引用了多少次，为0的时候进行清理
  • 标记清除
    • 将所有存储在内存中的变量都作上标记
    • 去掉环境变量中的标记，以及被环境变量所引用的变量的标记
    • 清除掉所有带标记的变量

如何判断是否内存泄漏

经验法则是，如果连续五次垃圾回收之后，内存占用一次比一次大，就有内存泄漏。这就要求实时查看内存占用。

常见的内存泄漏

  1. 全局变量
    • 可以在js头部加上’use strict’来避免意外的全局变量
    • 必须使用全局变量存储时，要在使用完之后将其设为null或重新定义
  2. 被遗忘的计时器或回调函数
    • 计时器停止时才会被回收

如何尽快的查到哪一行导致内存泄漏

使用chrome的memory作堆快照分析，两个快照进行比较。找到是什么对象在泄漏，然后去代码里找这个对象在哪些地方被使用了。

前端模块化

发表于 2020-05-14

为什么要进行前端模块化

使用模块可以将项目功能分离，便于维护，方便复用，以及避免污染全局变量。

模块化的几种规范

commonjs

  • 通过require来调用模块，module.exports来定义模块对外输出的接口，这样就可以使用模块暴露出来的公用方法/变量。
  • 采用同步进行加载，加载完才会执行后面的操作
    • 产生了AMD/CMD

AMD（Asynchronous Module Definition） - 异步模块定义

  • 代表： requirejs
  • 定义及使用
    • 定义模块define(id?, dependencies?, factory)
    • 调用模块require([dependencies], function(){})
  • 提前加载依赖模块，模块加载完毕后执行回调函数

CMD

  • 代表：seajs
  • 采用就近依赖的方法来加载模块，不在dependencies里依赖模块，在factory里就近加载依赖模块

es6 module

  • 使用import和export来进行模块间的调用
  • 提供了export deafult来作为默认输出

commonjs与es6 module的差异

最本质的区别在于前者对模块依赖的解决是动态的，而后者是静态的。

  • 动态：模块依赖关系的建立是发生在代码运行阶段；静态：模块依赖关系的建立是发生在代码编译阶段；
  • commonjs是同步加载，es6模块是编译时输出接口
  • commonjs输出的是值的拷贝，输出后模块的变化不会影响到这个值；es6模块输出的是值的引用，真正执行时再去取值
  • es6 module支持webpack的摇树（抖掉多余代码
    • 因为esm是静态编译的，而且import只能写在最顶端，所以可以根据这个作依赖分析，从而抖掉多余模块
