变量提升

使用var对name进行二次声明后，相当于在匿名函数内name这个变量已经被占用了，即在匿名函数内部发生了变量提升，所以一开始定义的name被内部的name覆盖，又因为变量提升只提升声明不提升初始化，所以在匿名函数的内部name为undefined，最后输出Goodbey Jack

this的指向

  • 第一个console，在运行时，有两层，一个是obj，另一个是obj.prop。如果一个函数中包含this，this指向它上一层的对象，即obj.prop，所以结果是3
  • 第二个console，运行时this指向的是window，this.fullname = window.fullname = 1

箭头函数的特性是它的this是上下文的this，在本段代码中，getFullname中的this就是prop对象中的this，在普通函数中，this是在函数调用时才被确定的，而在箭头函数中，this的指向在一开始就被确定了。

箭头函数的this是从作用域的上一层链继承的，在本段代码中，只有对象，所以this的指向不断向上，最终指向windows，输出两个1

一开始用了webstorm运行，运行出来的结果是undefined，这是因为在webstorm创建的js文件中，默认是nodejs文件。两个语言虽然语法相同，但是还是有很大区别的。粘贴到chrome的控制台之后，就都输出了1.

盒模型

盒模型分为IE盒模型和标准盒模型。

  • IE盒模型 = margin + content( border + padding + height/width )
  • 标准盒模型 = margin + border + padding + content( height/width )

如果设置IE盒模型height:100px，那么这100px中包括border + padding + height/width，即css中的height实际上设置的是content的高

同源策略

同源策略的限制

同源策略会使以下三种受到限制。

同源

同源指的是协议、域名、端口这个三元组一致。

http://toutiao.com和http://www.toutiao.com不同源，因为它们的域名不同。

CSRF

csrf是因为自动携带cookie而产生的安全性问题，为什么有同源策略还会出现这个问题呢？

因为同源策略对cookie的限制是无法读取，但是发请求的时候cookie是属于自动携带。所以即使是非同源，也是默认会携带cookie的。

如何限制csrf？

可以考虑用cookie的sameSite属性进行限制。

httpOnly

只能防止js读取cookie

css部分

font-size

最小值12px(chrome)

如何突破font-size的最小值？
  1. 使用transform:scale(0.75)，进行缩放
  2. 用户自行调整chrome://settings/fonts，这个方法和前端没关系

transform

transform会触发硬件加速，拥有这个属性的元素会被提升到复合层，不触发重排&重绘。

什么情况下会创建层
  • 3D 或透视变换(perspective transform) CSS 属性
  • 使用加速视频解码的元素，如<video>
  • 拥有 3D (WebGL) 上下文或加速的 2D 上下文的元素，如<canvas>
  • 混合插件(如 Flash)
  • 对自己的 opacity 做 CSS 动画或使用一个动画 webkit 变换的元素
  • 拥有加速 CSS 过滤器的元素，如CSS filters
  • 元素有一个包含复合层的后代节点(换句话说，就是一个元素拥有一个子元素，该子元素在自己的层里)
  • 元素有一个 z-index 较低且包含一个复合层的兄弟元素(换句话说就是该元素在复合层上面渲染)

css3硬件加速（GPU加速）

通过GPU进行渲染

触发硬件加速
  • transform
  • opacity
  • filter
  • will-change：哪一个属性即将发生变化，进而进行优化。e.g.will-change:transform
GPU加速的坑

会导致内存问题

GPU加速的优点

更快，动画会很流畅

浮动塌陷

解决方法：

  1. 兄弟元素clear:both
  2. 父级元素overflow:hidden
为什么父级元素设置overflow解决浮动塌陷

设置overflow之后会形成BFC，开始计算父级元素的高度，所以会被撑开。
